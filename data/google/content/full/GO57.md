Abstract

While program generation using Large Language Models (LLMs) dramatically improves software development productivity, ensuring the quality reliability (Robustness) and logical consistency of the generated code remains the biggest challenge. This paper proposes a comprehensive best practices framework to overcome the limitations of LLMs statistical inference and achieve high-accuracy program generation. This framework consists of a multi-stage approach centered on the rigorous structuring of input information, the enforcement of Chain of Thought (CoT) during the generation process, and independent quality auditing via AI-on-AI verification. This approach integrates AIs generation capability with human expert judgment, minimizing the risks of hallucination and rework in program generation.

1\. Introduction: The Paradigm of AI-Driven Development and Quality Challenges

The utilization of Generative AI accelerates the concept of Model-Driven Architecture (MDA), making the automated transformation from design models to implementation code a reality. However, because LLMs are inherently probabilistic models, they possess a fundamental vulnerability: the tendency to generate logical contradictions or misinformation (hallucinations) when creating program logic that requires strict formal logic or deterministic inference 1,.1To automatically generate high-quality programs, it is essential not merely to rely on natural language instructions (prompts) but to accurately guide the AIs inference and integrate an objective verification process for the generated artifacts. This paper proposes a framework consisting of the following six steps and discusses its technical validity and significance for quality assurance.

2\. Proposed Best Practices Framework

The proposed framework consists of two phases, Rigorous Planning and Input and Separation of Execution and Verification, strategically combining human judgment with AIs generative power

3\. Phase I: Rigorous Planning and Input (Steps ①–④)

The objective of this phase is to establish an environment where AI can infer the programs static structure and dynamic logic without misinterpretation and with consistency.

3.1. Enforcement of Structured Modeling and Ambiguity Exclusion (Step ①)

To enhance prompt accuracy (preventing judgment ambiguity), natural language instructions are avoided, and model development artifacts are directly utilized as input data.

・Rigorous Static Structuring: UML class diagram information is structured as JSON Schema or data dictionaries containing multiplicity, relationship types (aggregation/composition), and attribute data types 2, and fed into the LLM. This ensures the strictness of the generated codes data model and object references.

・Determinization of Dynamic Logic: State transition diagrams are input not as visual information but as a structured state table defining guards and actions. This compels the LLM to process deterministic logic based on the input logical expressions, rather than statistical inference.

3.2. Visualization of Thought Process (CoT) and Design Auditing (Steps ②, ③, ④)

CoT and XAI (Explainable AI) are integrated to prevent the LLMs inference from being a black box, keeping it under the control of the human architect.

・Procedure Clarification via CoT (Step ②): Before code generation, the AI is instructed to declare step-by-step procedures such as core class definition, design pattern application (e.g., Strategy Pattern), and interface implementation. This declaration acts as the Chain of Thought (CoT) for code generation, maintaining logical consistency and preventing logical leaps.

・Human Design Auditing (Step ③): The procedure proposed by the AI is reviewed and adjusted by a human designer. This is an essential gate for reliably injecting critical business requirements and organization-specific constraints (security, performance, etc.) before the generation process begins.

・XAI Verification and Grounding (Step ④): XAI is used to verify the rationale for the AIs proposal at each step. By utilizing RAG (Retrieval-Augmented Generation), the LLM is compelled to cite the specific sections of the official specifications or rule bases it referenced as evidence 1, proving that the AIs inference is based on formal definitions rather than statistical plausibility.1 Furthermore, the system must constantly check that invariants are not violated.

4\. Phase II: Execution, Verification, and Documentation Automation (Steps ⑤–⑥)

The objective of this phase is to independently guarantee the quality of the generated code while maximizing the efficiency of documentation and maintenance.

4.1. Incremental Generation and AI-on-AI Verification (Step ⑤)Generation and verification roles are separated to manage complex logic and overcome quality risks inherent to AI-generated code.

・Complexity Management through Incremental Generation: By dividing program generation into the defined steps of the procedure, the amount of context the LLM processes at once is limited, suppressing the occurrence of hallucinations.

・Independent Quality Auditing via AI-on-AI: A separate Generative AI (or an AI assigned a different persona) from the one that generated the program is tasked with creating and executing test programs. This is a highly rational independent auditing strategy to avoid the risk (limit of self-verification) that the program-generating AI inherits its own inference biases into the test design.

・Improved Coverage and Maintainability: The test generation AI has the ability to understand specifications and propose optimal test cases, enabling extensive coverage optimization. Furthermore, AI-driven testing frameworks are equipped with a self-healing function that automatically repairs test scripts to prevent failures due to UI changes or feature additions, significantly reducing the cost of test maintenance.

4.2. Quality Assurance and Documentation Synchronization (Step ⑥)

The final process after program generation is complete focuses on ensuring quality at the integration level and establishing a state where documentation is synchronized with the implementation.

・Ensuring Integration Quality through End-to-End Testing: To detect interface discrepancies and dependency mismatches between systems that might be missed by step-by-step testing, integrated end-to-end testing is executed after all steps are [completed.Live](https://www.linkedin.com/redir/redirect?url=http%3A%2F%2Fcompleted%2ELive&urlhash=OPou&trk=article-ssr-frontend-pulse_little-text-block) Document Generation via Reverse Engineering: Reverse-engineering specifications and technical documents from the generated program (code) in natural language is performed. This fundamentally resolves the traditional development problem of design document divergence from implementation code, ensuring the documentation is maintained as a live document that is always synchronized with the implementation. This process is a highly efficient method utilizing the AIs proficiency in natural language generation to ensure documentation reliability.

5\. Conclusion

The Best Practices for High-Quality, High-Accuracy Program Generation Using Generative AI framework proposed in this paper is a highly effective strategy for solving the quality reliability problem in AI-driven development through structured input and process control.

Specifically, the improvement of prompt accuracy through the rigorous structuring of UML models (Step ①), the visualization of the AIs thought process via CoT (Steps ②, ③), and the incremental and independent verification by AI-on-AI (Step ⑤) comprehensively guarantee the functional accuracy, efficiency, maintainability, and testability of the AI-generated code.

Moving forward, integrating this framework into the actual development pipeline and linking it with external formal verification tools 1 will enable the AI to function not merely as a coding assistant but as a highly reliable automated auditing system for critical system design, which is the roadmap for further quality enhancement.
