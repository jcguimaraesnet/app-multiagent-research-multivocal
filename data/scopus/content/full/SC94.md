- **ABSTRACT** Our research focuses on the intersection of artificial intelligence (AI) and software development, particularly the role of AI models in automating code generation. With advancements in large language models like ChatGPT, developers can now generate code from natural language prompts, a task that traditionally required significant manual input and expertise. AI-generated code promises to boost productivity by enabling faster prototyping and automating repetitive coding tasks. However, as these models are increasingly adopted in real-world applications, questions surrounding their efficiency and code quality become critical. This research investigates ChatGPT-4o, a state-of-the-art language model, and its ability to generate functional, high-quality code in different programming languages. By comparing performance between Python and Java, the study seeks to shed light on AI’s capabilities and limitations in code generation, addressing not only functional correctness but also broader software engineering concerns such as memory usage, runtime efficiency, and maintainability. The study addresses key questions related to the performance, code quality, and error management of AI-generated code by analyzing solutions for 300 data structure problems and 300 problems from the LeetCode platform. The findings reveal notable performance differences between the two languages: Java demonstrated superior runtime performance, particularly for medium and hard problems, while Python exhibited better memory efficiency across all complexity levels. The research also highlighted significant gaps in code quality, with both languages showing deficiencies in documentation and exception management. This study contributes to the literature by offering a comprehensive cross-language analysis of ChatGPT-4o’s programming capabilities, addressing a gap in the evaluation of AI-generated code performance.

For instance, AI-assisted programming tools enable developers to interact with code through natural language commands, making the coding process more intuitive and efficient [1], [2]. The early generations of AI-generated code, particularly from models like ChatGPT, present challenges that warrant thorough investigation. A primary concern is the correctness of the generated code. AI-based models, trained on vast datasets from various sources, may produce code that contains errors or fails to adhere to best practices. This approach simulates real-world programming conditions, offering insights into how well ChatGPT-4o performs when faced with practical coding problems.

### **III. LITERATURE REVIEW**

While ChatGPT is designed to understand the context surrounding code segments, it risks making errors and proposing unnecessary alterations. Despite these challenges, its overall impact on programming has been positive. However, human oversight is still necessary to review the proposed changes and determine their usability. DePalma et al. tested ChatGPT3.5’s refactoring capabilities using various prompts aimed at improving specific quality attributes such as performance, complexity, and coupling. Bucaioni et al. [22] evaluated the capabilities of ChatGPT3.5 in C++ and Java by tackling a diverse set of problems solvable through dynamic programming techniques. The code generated by ChatGPT-3.5 was submitted to LeetCode, which served as a robust platform for compiling and testing the code against standard test cases. ChatGPT’s solutions were compared with those provided by human programmers, evaluating efficiency and resource utilization. The findings revealed that while ChatGPT-3.5 is effective at solving simple and moderate programming tasks, it struggles with more complex challenges. It is also important to note that the generated code, though functional, does not consistently meet standard quality requirements.

The study concluded that while ChatGPT-3.5 can be helpful in suggesting security-related improvements, it still requires oversight to ensure safe practices.

The results were compiled into tables documenting the feedback, which included a list of identified strengths, weaknesses, and recommended changes. The study highlights the potential of ChatGPT-3.5 to help enhance execution speed, readability, and maintainability, although paradoxical suggestions were observed in cases where reducing character count conflicted with readability improvements.

### _B. PERFORMANCE AND QUALITY METRICS_ 

In our experiment, we will employ several performance metrics to evaluate the effectiveness of the code and its associated operations. Key metrics include execution speed and memory usage. Execution speed refers to the time it takes for a program to complete its tasks, measured in milliseconds. This metric is crucial for assessing code efficiency, as faster execution is often a critical requirement. In contrast, memory usage measures the amount of system memory consumed by the program during execution. This metric is essential for understanding how resource-intensive the application is, as excessive memory usage can lead to system slowdowns, particularly in environments with limited hardware resources. 

In addition to performance metrics, the study will consider potential errors that may affect the code’s reliability and robustness. Two primary categories of errors will be assessed: compile errors and runtime errors. Compile errors occur during the compilation phase and are typically related to 

syntax issues, type mismatches, or missing dependencies. These errors prevent the program from running altogether and are generally easier to identify and fix prior to execution. Conversely, runtime errors occur while the program is running and may be caused by unforeseen conditions. These errors can be more challenging to detect and resolve, as they depend on the system’s state and the data provided during execution. 

Finally, we will analyze the quality of the code based on several factors. The quality of documentation is crucial for maintaining and extending the code, as well-documented code is easier to understand, debug, and collaborate on. We will also pay attention to unused variables, which not only clutter the code but can also consume unnecessary memory. Additionally, proper exception-handling mechanisms will be evaluated, as robust error handling ensures that the program can recover from unexpected events, minimizing potential disruptions in execution.

### _A. PERFORMANCE ASSESSMENTS_ 

The performance metrics presented in this section are derived using LeetCode’s benchmarking suite. This platform evaluates runtime and memory efficiency by executing code against a set of standardized test cases.
