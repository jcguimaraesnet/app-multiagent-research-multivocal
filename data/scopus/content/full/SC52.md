## **ABSTRACT** 

**Objective:** The task of code generation aims to transform natural language descriptions into corresponding target code. Among the various approaches, syntax-aware code generation has emerged as a significant approach that strives to generate code by directly modeling the underlying syntactic rules. However, existing works typically adopt an autoregressive approach to sequentially generate each abstract syntax rule, which inevitably neglects the rich structural semantic information inherent within the syntax rules. To address this issue, we propose an enhanced architecture of structure semantics based on Graph Neural Network for code generation. 

**Methods:** Our approach explicitly models the internal structure of syntactic rules by treating them as graph data, thereby enabling the extraction of deeper structural semantics. Furthermore, we jointly model both the sequential semantics and structural semantics of syntactic rules, effectively addressing the limitations of solely sequence-based approaches in capturing the inherent structural semantics of code. 

**Results:** Experimental results on two widely used code generation datasets demonstrate that the proposed model consistently outperforms strong baselines, with gains of up to 2.14 BLEU points and 2.02 CodeBLEU points, highlighting the effectiveness of our structural-semantic modeling approach for code generation.

Inspired by advancements in natural language processing (NLP), researchers have increasingly applied deep learning models to code generation tasks, resulting in remarkable performance improvements. Overall, existing code generation approaches based on deep learning can be broadly categorized into three main classes [3]. The first class utilizes large-scale pretrained models such as GPT [4]. Beyond scaling, recent studies further incorporate post-processing mechanisms to refine model outputs, aiming to enhance code correctness and overall generation performance [9, 10]. The second class focuses on retrieval-based approaches, where relevant code segments or textual descriptions are retrieved from a database to facilitate the code generation process [11–13]. In contrast to the above approaches that rely heavily on large-scale parameters or external data, syntax-aware code generation approaches primarily focus on modeling the abstract syntax tree (AST) of the target code, rather than directly predicting raw code tokens [14]. By leveraging the intrinsic and stringent syntactic constraints inherent in code, syntax-aware approaches have not only effectively enhanced model performance, but also ensured the syntactic correctness of the generated code. Typically, these approaches adopt sequence-to-tree (seq2tree) models and have demonstrated highly effective performance in generating code for a variety of tasks [15–18]. 

Despite the promising results demonstrated by syntax-aware approaches in code generation tasks, their performance may be constrained by the inherent limitations of their modeling paradigm. Specifically, the AST, which serves as the underlying representation of code, contains rich structural properties that could prove crucial for accurate code generation [19]. However, existing approaches primarily rely on sequential, autoregressive generation of abstract syntax rules, which may overlook the rich structural and semantic information inherent in the AST. As a result, there is a need to explore modeling techniques that can better harness the structural information present in the AST, potentially leading to further improvements in the performance of syntax-aware code generation.

## **3.1 | Overview** 

In the code generation workflow depicted in Figure 2, our approach consists of three main stages: data preprocessing, model training, and code generation. 

In the data preprocessing stage, we address the challenge of the unbounded vocabulary space, particularly due to the diverse variable names derived from NL descriptions. To mitigate this issue, our preprocessing process aims at replacing the irregular variable identifiers with predefined canonical names. Specifically, variables are renamed as _var_0_ , _var_1_ , etc., while string literals are relabeled as _str_0_ , _str_1_ , etc. This systematic renaming procedure serves to constrain the vocabulary space, consequently facilitating more efficient learning of the lexical distribution. Furthermore, to enhance the efficacy of supervised learning in subsequent stages, we endeavor to fully exploit the informative structural characteristics inherent in code snippets. Specifically, we formalize a given AST with _N_ actions as a directed acyclic graph (DAG)  = ( _𝑉, 𝐸_ ), where _𝑉_ represents the node set and _𝐸_ denotes the edge set. The node _𝑣𝑖_ ∈ _𝑉_ indicates one grammar rule action, while the edge ( _𝑣𝑖, 𝑣𝑗_ ) ∈ _𝐸_ encodes the associative relationship between node _𝑣𝑖_ and node _𝑣𝑗_ . Subsequently, an adjacency matrix  ∈ ℝ<sup>_𝑁_×</sup><sup>_𝑁_</sup> derived from the DAG is used to captures the topological information. An element of  indicates the presence (or absence) of a directed edge from node _𝑣𝑖_ to node _𝑣𝑗_ , where _𝑎ij_ = 1 if the edge exists, and _𝑎ij_ = 0 otherwise. Compared to the conventional preorder or postorder node traversal approaches, utilizing the AST adjacency matrix facilitates a more direct and explicit capture of structural information. Additionally, a preorder traversal approach similar to traditional 

approaches is also adopted to obtain the AST node sequence, enabling sequential information modeling in subsequent stages. 

In model training, we incorporate both the action sequence and the AST adjacency matrix as input to our SSECG model. Through the encoding-decoding process, the model extracts sequential semantic information and structural semantic information, serving as complementary semantic signals. The integration of these two complementary modalities of information enables our model to effectively learn the transformation from NL to the target AST through a supervised learning paradigm. The details of this stage will be elaborated upon in Section 3.2. 

During code generation phase, the trained SSECG model is employed to perform inference, mapping NL description to AST. The target code can subsequently be derived by converting the generated AST using a parser constructed during the data processing stage. Additionally, retrieval-augmented approaches can be integrated into the framework to further enhance the inference performance from NL to AST.

## **4.1 | Dataset** 

We conduct our experiments on two general-purpose code generation benchmark datasets: DJANGO [31] and CONALA [32], both of which are classic experimental datasets in the field of code generation. As shown in Figure 4, the two datasets both include annotated NL descriptions and their corresponding code snippets. Although the code snippets in both datasets are only at line-level granularity, the complex structure within the code snippets poses a significant challenge to deep learning-based code generation models. 

DJANGO [31] is a collection of python source code extracted from the Django web framework. It has totally 18,805 lines of codes and each of which is annotated with an NL utterance. The source code in this dataset covers a wide variety of real-world python use cases, including exception handling, function calls, conditional statements, and loop controls, among others. CONALA [32] contains a larger corpus of 600 k NL-code pairs, of which 2879 examples have been manually cleaned up. Each NL-code pair in CONALA consists of an NL description and its corresponding python code from _StackOverflow_ . Compared to DJANGO dataset, the NL queries in CONALA are derived from a broader range of problem requirements encountered in real-world development scenarios. Consequently, the complexity of the queries and corresponding code is substantially elevated, significantly augmenting the challenges of the code generation task.

## **4.3 | Baselines** 

We compare our model against the following baselines, which cover a range of techniques including syntax-aware approaches, retrieval-augmented approaches, and approaches based on large-scale pretrained models: 

- _TRANX_ [14]: It employs a transition system based on abstract syntax description language (ASDL) to map NL descriptions to abstract syntax trees through a sequence of tree construction actions. 

- _External Knowledge With API_ + _Reranking_ [13]: It enhances the performance of TranX model by pretraining on two external knowledge bases and utilizing additional hypothesis reranking. 

- _BART Base_ [35]: It utilizes BART as an unsupervised multimodal learner for the code generation task. 

- _TAE_ [36]: It enhances performance by taking advantage of the CONALA mined examples from _StackOverflow_ . 

- _BerTranX_ [15]: It introduces a vocabulary replacement component based on syntactic constraints, lexical preprocessing, input representation, and copying mechanism, effectively enhancing the upper bound of code generation performance. 

- _CodeGPT_ [5]: A pretrained decoder-only language model that can handle diverse code-related tasks, including text-to-code generation and code-to-text summarization. 

- _CodeT5_ + [6]: A flexible encoder-decoder language model that leverages diverse pretraining on multilingual code to achieve state-of-the-art performance across code tasks.

## **5.2 | Qualitative Analysis** 

To provide deeper insights into the effectiveness of our explicitly AST modeling mechanism, we present a multi-faceted qualitative evaluation and introduce how this mechanism facilitates the generation of AST. 

Figure 5 is a concrete example showcasing how our explicit structural decoding mechanism provides complementary information to enhance the sequential decoding process. Specifically, the NL description _limit float_ 3.14159 _to two decimal points_ is utilized for training to generate the target code (‘%.2 _f_ ’ % 3.14159). The sequential decoding process obtains the decoded features of AST by predicting the AST nodes in a step-by-step manner, effectively capturing the sequential structure of the code. In contrast, the structural decoding process captures deep-seated relational features and structural dependencies between the AST actions on the basis of GNN encoding, providing a complementary modeling perspective. For instance, at time step _𝑡_ 4, the structural feature of “ _Mod_ ” action is fused with the sequential feature, enabling the model to more effectively learn the deep-level structure of AST during training. The integration of sequential and structural decoding enables our model to jointly exploit both the sequential and hierarchical aspects of the AST, leading to a more comprehensive and coherent code generation capability. 

To further validate our opinion regarding the benefits of integrated sequential and explicit structural decoding, we visualize the attention weights between AST nodes during the sequential decoding and the structural decoding, respectively. As shown in Figure 6, we can observe that the sequential decoding process focuses more on parent or sibling history nodes, reflecting its modeling of the code’s sequential structure. In contrast, the structural decoding process pays more attention to related nodes at the same hierarchical level, demonstrating its understanding of the AST’s hierarchical relationships. Integrating these two distinct attention mechanisms helps the model better exploit both the sequential and hierarchical characteristics of the AST, enabling a more comprehensive grasp of the tree’s structure and semantics. 

As illustrated in Figure 7, we typically choose a test case from CONALA dataset at random, and evaluate the abstract syntax tree output of our model and BerTranX on this example. We find that BerTranX makes an incorrect prediction at the fifth time step, resulting in an incorrect code generation result, whereas our model is able to predict correctly. This discrepancy can be attributed to the fact that certain grammar keywords, such as the “ _Call_ ” keyword and “ _Subscript_ ” keyword, exhibit structural similarity, making it challenging for sequential decoding models to differentiate between them. By explicitly modeling the hierarchical structure of various grammar keywords through our structural decoding mechanism, our model is capable of distinguishing different grammar 

keywords at a deeper level, and thus makes more accurate predictions. 

## **5.3 | Ablation Study** 

To investigate the contribution of the different components of SSECG, we conduct a detailed ablation study and report the results in Table 4. Specifically, we provide the performance of the following ablated versions of our model on CONALA and DJANGO in the same configuration: 

- SSECG w/o structural decoding. It is only equipped with our proposed structure-aware loss strategy. 

- SSECG w/o structure-aware loss. This simplified model only contains our proposed AST structural decoding module. 

Both the AST structural decoding and the structure-aware loss strategy aim at introducing explicit structural information during model training, but they approach this from different perspectives. The AST structural decoding focuses on modeling AST from a graph-based perspective, capturing the relational features and dependencies between the AST nodes. In contrast, the structure-aware loss strategy focuses on directly guiding the model to consider the structural semantic information of AST during model training. 

From the results presented in Table 4, we observe that CONALA and DJANGO datasets exhibit different dependencies on the two structural information enhancement strategies. The NL-Code pairs in CONALA dataset are of a query-code form, where the extractable semantic information about the code structure is generally stronger. Thus, the effectiveness of the structural decoding is more obvious for this dataset. In contrast, the DJANGO dataset is closer to a pseudo-code form, so the model tends to focus more on learning the direct mapping relationship between the NL and the AST during the training process. Consequently, the effectiveness of the structure-aware loss strategy is more obvious for DJANGO dataset. These results demonstrate the effectiveness of our two AST structural information enhancement strategies. By leveraging both the structural decoding mechanism and the structure-aware loss strategy, our model is able to exploit the inherent structural properties of AST in a comprehensive manner.

## **7 | Related Work** 

As described previously, the proposed SSECG architecture employs GNNs to explicitly model the structural semantics of AST, thereby facilitating the incorporation of deeper structural semantics during the code generation process. In this section, we first present an overview of recent advancements in the field of code generation. Subsequently, we review representative works that have explored the application of GNNs in effectively capturing and exploiting structural information across various domains. 

## **7.1 | Code Generation** 

In the software development process, code generation plays a crucial role in improving productivity and reducing development time. Traditionally, code generation has been approached through template-based or rule-based techniques, which often rely on predefined patterns and structures [38, 39]. Nevertheless, these approaches exhibit significant limitations when confronted with the complexity and heterogeneity inherent in real-world programming tasks. More recently, data-driven code generation approaches based on machine learning have emerged as a promising solution to address the shortcomings of traditional techniques. These approaches leverage NL-code pairs to learn the underlying patterns and semantics of code, enabling the generation of target code snippets that are tailored to specific NL descriptions. 

Among the data-driven code generation approaches, syntax-aware code generation has gained significant attention. These techniques are intended to generate code that adheres to the syntactic rules and structures of the target programming language. TranX [14], for instance, utilized transformation action sequences to incrementally construct a semantic parsing tree, indirectly mapping NL descriptions to the target code. Treegen [16] designed a novel tree-based neural architecture that aims to address the issue of long-dependency problems in 

code generation tasks. BerTranX [15] introduced a vocabulary replacement component based on syntactic constraints, lexical preprocessing, input representation, and copying mechanisms, effectively enhancing the upper bound of performance in code generation. Further advancements include the work of Sun et al. [40], who proposed a grammar-based structural convolutional neural network (CNN) and generated programs by predicting the syntax rules of a programming language. Wei et al. [17] introduced a dual training framework that simultaneously learns the duality between code generation and summarization tasks, significantly improving both. Contextor [18] employed a synchronous bidirectional decoder to generate AST tokens in two different traversal orders simultaneously, enabling the model to synthesize both previous and future information. 

In addition, there exists another category of retrieval-augmented code generation approaches. These approaches endeavor to address the code generation task by identifying and retrieving the most relevant code snippets from a large corpus, based on the input NL description. Recoder [12] utilized a vast repository of code examples to enhance the generation of code snippets by retrieving and adapting relevant existing code segments. Xu et al. [13] proposed a model-agnostic data augmentation approach, leveraging external knowledge base data for pretraining the model, followed by fine-tuning on manually annotated CONALA dataset. RRGcode [11] employed a deep hierarchical search approach and optimized the ranking of initial search codes through three key stages: retrieval, reordering, and generation. This reduces the risk of errors in copying search corpora and improves the ability to generate higher quality and more reliable code. KNN-TRANX [37] improved neural code generation by using token-level retrieval in smaller, task-specific datastores and leveraging syntax constraints to reduce noise. 

More recently, large-scale pretrained models have also been explored as a promising direction for code generation, leveraging the representational strength of transformer architectures [41], to better capture the complex relationships between NL and code. One such approach is CodeT5+ [6], which is an enhanced version of CodeT5 [42] with a larger parameter scale to provide higher performance and more accurate code understanding. Extensive experimental results have demonstrated significant improvements achieved by CodeT5 in both code understanding and code generation tasks. PLBART [7], built upon the BART architecture [43], is a unified pretraining model designed to improve both program understanding and generation by effectively handling noisy or corrupted input sequences. CodeGPT [5], a decoder-only model based on GPT-2, is tailored for language generation tasks, particularly the synthesis of code snippets from NL prompts or partially written code. ExploitGen [44] extends CodeBERT [45] with a template-augmented framework that integrates template information with NL semantics to improve the quality of automatically generated exploit code. Furthermore, large language models (LLMs) such as Codestral and CodeGemma have significantly advanced code generation by training on massive and diverse code corpora [46, 47]. Despite their strong performance, these pretrained-based approaches typically entail substantial model complexity and computational cost. 

Overall, existing studies have primarily focused on three directions: enhancing model architectures to better capture the

sequential dependencies in code, integrating external knowledge through retrieval mechanisms, and scaling models in terms of parameters and training data. Despite these advances, explicit modeling of the inherent structural properties of code remains underexplored in code generation tasks, even though it can significantly contribute to more accurate understanding and generation [48]. To address this limitation, we propose a code generation method that simultaneously leverages both sequential and structural information, aiming to achieve more precise code understanding and generation even in small-model scenarios. 

## **7.2 | Graph Neural Networks** 

Graph represents a ubiquitous data structure encountered across diverse domains, ranging from social networks to molecular structures. However, the inherent irregularity and non-Euclidean properties of graph data pose significant challenges for traditional neural network architectures such as CNN [49] and Recurrent Neural Networks (RNN) [50], which are primarily designed for processing grid-structured or sequential data. To address these limitations, GNN [20, 21] have emerged as a powerful paradigm for modeling and analyzing structural data. By explicitly encoding relational information and topological structures inherent in graphs, GNN demonstrates superior performance compared to conventional neural networks across a wide spectrum of tasks involving structural inputs, including but not limited to node classification and link prediction. 

Graph Convolutional Network (GCN) [51] is the first seminal GNN, which extended the concept of convolution operations from regular grid-structured data to irregular graph data, enabling effective feature propagation and aggregation across graph nodes. Building upon the foundations of GCN, GAT [29] further advanced the capabilities of GNN by incorporating attention mechanisms to dynamically weigh the importance of neighboring nodes during the feature aggregation process. This attention-based approach enables the model to focus on the most relevant parts of the graph, leading to more informative node representations and improved performance on various graph-based tasks. Beyond these pioneering architectures, the field of GNN has continued to evolve, with researchers proposing a range of novel models to address specific challenges and enhance graph representation learning. For instance, Graph Isomorphism Network (GIN) [52] was designed to better capture subtle structural differences between graphs, enabling more expressive node embeddings. The ability of GNN to capture the inherent structure and dependencies in graph data has made it a versatile and highly influential tool in the field of machine learning and data mining, enabling successful applications across a wide range of domains. Nevertheless, due to the intrinsic complexity of graph generation tasks, graph-based modeling for code generation remains highly challenging, highlighting the need for further research [27]. 

## **8 | Conclusion** 

In this work, we have presented SSECG, an enhanced architecture of structure semantics for code generation that advances the state-of-the-art in syntax-aware approaches. Unlike previous 

works that primarily rely on sequential modeling of the AST, SSECG comprehensively leverages both the sequential and hierarchical characteristics inherent in code. By explicitly modeling the hierarchical relationships, compositional structures, and semantic dependencies inherent in AST, our approach significantly empowers previous seq2tree baselines. This explicit structural modeling, combined with the effective fusion of sequential information, enables SSECG to better capture the rich semantics encoded in the code’s underlying representation. Moreover, we presented a bi-modal information modeling framework based on SSECG, which integrates the sequential and structural perspectives for comprehensive code generation. 

Experimental results on two widely used code generation datasets demonstrate the superior performance of SSECG in code generation tasks. We note, however, that these datasets are limited to line-level granularity and single-file scope, lacking multi-file relationships and realistic software development scenarios. Despite these limitations, they provide valuable benchmarks for evaluating code generation models. In future work, we plan to explore domain-specific applications and utilize datasets that incorporate multi-file dependencies and more complex code structures, enabling a more comprehensive evaluation of the proposed approach. Additionally, we will investigate the integration of our structure-semantic enhancement strategy with advanced technologies, such as transformer-based architectures [41] and reinforcement learning [53], to further improve the accuracy and applicability of code generation systems.
