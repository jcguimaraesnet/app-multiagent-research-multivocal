- **ABSTRACT** Code duplication, commonly known as code cloning, is a persistent challenge in software development. While reusing code fragments boosts productivity, excessive cloning poses challenges to maintenance and elevates the risk of bugs. Therefore, integrating code clone detection into the development process is crucial. The extensive code-related knowledge inherent in Large Language Models (LLMs) renders them high-potential candidates for addressing diverse software engineering challenges. However, the effectiveness of LLMs in the specific task of code clone detection requires precise evaluation. This paper proposes an innovative methodology leveraging few-shot instruction-tuned GPT-3.5 Turbo and GPT-4 to detect code clones across all types, focusing on complex clones (Type-3 and Type-4). Unlike conventional approaches confined to specific language pairs or tasks, our method employs versatile language models, showcases generalization strengths for semantic understanding, and leverages instruction tuning with few-shot inference for task-specific adaptability in code clone detection. A conversational dataset was crafted from BigCloneBench for instruction tuning, enhancing task alignment and performance. This study evaluates the proficiency of LLMs in identifying code clones, analyzing the impact of instruction tuning, and assessing the efficiency across various clone types. Experimental results demonstrate these models achieving competitive performance against existing tools for overall and complex clone detection. Integration into an Integrated Development Environment (IDE) enables real-time detection and automated refactoring, bridging the gap between theoretical advancements and practical usability. This work highlights the potential of generalized LLMs setting a new standard in a field traditionally dominated by specialized tools and demonstrates their adaptability for complex challenges in code analysis and maintainability.

### _B. WHY USE GENERATOR-BASED MODELS FOR CODE CLONE DETECTION?_ 

In this study, we utilize GPT-4 and GPT-3.5 Turbo generator-based LLMs known for their generalization capabilities and robust performance in natural language tasks. Unlike classifier-based models, which are trained specifically for downstream tasks, generator-based models like GPT are trained using an auto-regressive objective, allowing them to predict the next token in a sequence based on the context. This training paradigm equips GPT with a deep understanding of sequential dependencies and contextual relationships, making it particularly effective for tasks requiring semantic analysis, such as code clone detection. 

Additionally, classifier-based models often struggle to detect clones that differ significantly in syntax but share identical functionality, a challenge GPT overcomes through its extensive training on diverse datasets, including programming languages, which enables it to capture complex patterns and recognize functional similarities between code snippets. This ability to generalize across varied contexts is crucial for detecting semantic clones, where code fragments may differ in syntax but perform the same function.

### _A. TRANSFORMERS_ 

Transformers have emerged as a pivotal topic in modern discussions within deep learning. Proposed by a team of researchers at Google in their seminal paper ‘‘Attention is all you need’’ [11], the term ‘‘Transformer’’ signifies a model’s extensive use of the Self-Attention Mechanism. This mechanism enables the model to focus on various parts of the input sequence, departing from the sequential processing of earlier models like Recurrent Neural Networks (RNNs). Transformers employ a sophisticated architecture comprising encoder and decoder layers, multi-head attention, and feedforward networks [12]. 

Models, such as BERT, GPT, and T5, are transformers that efficiently decipher relationships between words in a sentence or text sequence through parallel processing. They have significantly impacted diverse applications in natural language processing, including language translation, sentiment analysis, and text summarization [13]. 

### 1) OVERVIEW OF TRANSFORMER ARCHITECTURE COMPONENTS 

The transformer architecture is distinguished by its innovative components, each contributing to its efficiency and effectiveness in processing sequential data, especially in natural language processing tasks. Figure 1 provides a high-level overview of transformer architecture elements [11]. One key element is the Encoder-Decoder Structure, a fundamental design that enables the transformer to handle input and output sequences effectively. The encoder processes the input sequence, while the decoder generates the corresponding output sequence, making it particularly well suited for sequence-to-sequence tasks such as machine translation. 

To address the challenge of capturing the sequential order of tokens, the transformer incorporates Positional Encoding. This incorporation is crucial for maintaining the temporal relationships between elements in the sequence, ensuring that the model can understand the input’s sequential nature. The heart of the transformer’s capability lies in the selfattention mechanism, which replaces traditional recurrent and convolutional layers. This mechanism allows each element in the sequence to focus on different parts, capturing long-range dependencies. The introduction of Multi-Head Attention further enhances this capability, as multiple parallel selfattention mechanisms, or ‘‘heads’’ collectively contribute to a richer representation of relationships within the data. 

The Feedforward Neural Network component follows the self-attention mechanism, introducing non-linearity to the model. This neural network processes each position independently, capturing complex patterns in the data. Additionally, Layer Normalization and Residual Connections are employed to stabilize training. Residual connections, specifically, facilitate the flow of gradients through the network, aiding in the training of deep models. Training the transformer involves utilizing a Sequence-to-Sequence Objective Function where the model predicts the next element in the sequence. This objective is often complemented by additional objectives, such as attention regularization, to enhance the learning process. 

In summary, the architecture components of the transformer - the Encoder-Decoder Structure, Positional Encoding, Self-Attention Mechanism, Multi-Head Attention, Feedforward Neural Network, and the integration of Layer Normalization and Residual Connections - collectively contribute to its ability to capture intricate patterns, dependencies, and long-range relationships within sequential data, making it a foundational model in various applications, particularly in natural language processing.

### _C. LLM FINE-TUNING_ 

Within the domain of LLMs, fine-tuning stands as a pivotal process, serving as the gateway to refining pre-trained models on specialized datasets and optimizing their performance for specific tasks or domains [19]. Before fine-tuning, various techniques were employed directly within the user prompt to enhance the model’s output and customize it according to user needs. Two significant methods include in-context learning and zero, one, and few-shot inference [20]. In-context learning enhances the prompt by integrating specific task examples directly within it, providing LLM with a clear blueprint of the intended objectives [21]. Zero-shot inference entails inputting data to the prompt without additional examples [22]. If zero-shot inference is inadequate, methods such as ‘one-shot’ or ‘few-shot inference’ can be employed. These methods entail incorporating one or more complete examples within the prompt, thereby assisting smaller LLMs in achieving improved performance. 

LLM fine-tuning involves refining pre-trained models on smaller, specific datasets to enhance performance in particular tasks or domains. Techniques such as in-context learning, zero/one/few-shot inference, and general fine-tuning are crucial components of this process. The proposed approach serves to bridge the disparity between general pre-trained models and the unique requirements of specific applications, thereby guaranteeing a closer alignment between the language model and human expectations. 

### _D. LLM INSTRUCTION FINE-TUNING_ 

One approach employed to enhance a model’s performance across a range of tasks is through the process of Instruction Fine-Tuning. It involves updating pre-trained language models using labeled data specific to a task. Unlike unsupervised methods, this process employs checked data. During Instruction Fine-Tuning, the model is provided with a dataset of labeled examples formatted as prompt-response pairs [23]. The model’s weights are then adjusted based on this supervised learning approach, resulting in a better alignment with specific tasks. This iterative process ensures that the LLM continues to evolve and adapt to diverse tasks, enhancing its versatility and practical applicability.

### _E. CODE CLONING_ 

Code cloning refers to the practice of duplicating code fragments within or across software systems, often resulting from copy-paste programming or deliberate code reuse. While code cloning can accelerate development in the short term, it introduces significant long-term challenges, including increased maintenance costs, higher bug risks, and reduced system efficiency. Consequently, identifying and addressing these clones is critical for ensuring software quality and maintainability. 

Code clones are broadly classified into four main types based on their similarities and differences [24]: 

- **Type-1 (Exact Clones)** : These are identical code fragments except for variations in whitespace or comments. They are the simplest form of clones and are typically the easiest to detect. 

- **Type-2 (Renamed Clones)** : These clones retain identical code structures but differ in naming conventions, such as variable or function names. Detecting Type2 clones requires models that can understand context beyond simple token matching. 

- **Type-3 (Near-Miss Clones)** : These code snippets exhibit structural similarity but may include minor modifications, such as added, removed, or altered statements. Detecting Type-3 clones requires a deeper understanding of code structure and semantics. 

- **Type-4 (Semantic Clones)** : These clones represent the most challenging type, as they perform the same functionality but are implemented with different syntax and structure. Detecting these clones requires an advanced understanding of the functional intent of the code. 

To evaluate the performance of clone detection models, researchers commonly use benchmarks like BigCloneBench [25], a dataset that provides labeled examples of all clone types. This dataset serves as a comprehensive resource for testing models across varying levels of similarity, from exact matches to complex semantic equivalence.

### **III. LITERATURE REVIEW** 

Identifying and mitigating code clones is crucial for enhancing software reliability and facilitating efficient development processes. While traditional methods of code clone detection often rely on handcrafted features and specific heuristics, the evolving landscape of software development necessitates more sophisticated approaches. Recent advancements in deep learning have shown promise in addressing the nuances of code clone detection. As our proposed approach resides within the domain of deep learning techniques, this section outlines the evolution and diversity of clone detection methods over the years, with particular emphasis on deep learning approaches. 

Li et al. introduced CCLEARNER in 2017, a token-based clone detection approach that utilizes deep learning [26]. This approach deviates from previous methods, which were specific to certain types of clones. CCLEARNER employs deep learning on the method-level, using both code clones and non-clones as positive and negative examples to train its model. It trains a binary class classifier by deducing commonalities and potential variations between the peers of known clone method pairs. Subsequently, it extracts similarity vectors for both types of examples and feeds them to a deep neural network (DNN). CCLEARNER then utilizes this binary classifier to compare methods pair-bypair within a given codebase, effectively identifying clones. The findings of the paper demonstrated promising results, as CCLEARNER achieved competitive clone detection effectiveness at a low time cost compared to approaches that did not leverage deep learning. 

Zhao et al. presented CDLH (Clone Detection with Learning to Hash) in 2017, a framework designed to detect functional clones in software [27]. It is a comprehensive deep feature learning framework that uses AST-based Long Short-Term Memory (LSTM) to derive representations of code fragments and hash function parameters. It enhances the comparison of code fragments as it takes into account both the lexical and syntactical data of source codes. By addressing the limitations of traditional methods that rely on handcrafted features based on lexical or syntactical information, CDLH utilizes supervised deep learning, where the learning process is guided by labelled examples of code. This approach formulates clone detection as a supervised learning to hash problem, allowing code to be transformed into hash codes for rapid functional similarity assessment. Experiments on software clone detection benchmarks demonstrated that CDLH surpasses current state-of-the-art approaches in identifying functional clone pairs in addition to its applicability to the other types of clones. 

Zhang et al. proposed a new method in 2019 for Type-3 code clone detection [28]. They start by using tree embedding to generate a vector for each intermediate node in the Abstract Syntax Tree (AST), which captures the structural information of the AST. These node vectors are then combined to form a tree vector using a lightweight method. The key step involves 

comparing the tree vectors using Euclidean distances to detect code clones. They developed a tool called TECCD based on this approach and tested it using the BigCloneBench [25] as well as seven other large-scale Java projects. The researchers opted for a lightweight embedding method to derive the AST vector from node vectors, which proved to be more efficient than RNN-based methods. Importantly, unlike existing techniques that rely on project-specific language to train deep learning models, their method does not depend on lexical information during the learning process, allowing the trained node vector representation to be broadly applicable across different projects. The results showed that this approach achieves high accuracy and recall, outperforming existing methods. 

In the same year, Chen et al. introduced Tree-Based Convolution for Clone Detection (TBCCD) for detecting semantic code clones using deep learning [29]. This method employs tree-based convolution on token-enhanced ASTs in order to capture both structural and lexical information from code fragments. Unlike previous methods, this approach does not require transforming an AST into a complete binary tree and demonstrates superior effectiveness and efficiency compared to the state-of-the-art CDLH approach. A key feature of this method is the introduction of a new embedding technique called Position-Aware Character Embedding (PACE). PACE represents each token as a position-weighted combination of characters one-hot embeddings. The TBCCD uses a tree-based Convolutional Neural Network (CNN) classifier, which processes pairs of code fragments simultaneously with shared parameters, focusing on minimizing the cosine similarity for non-clone pairs and maximizing it for clone pairs. This deep learning approach has demonstrated an improvement in detecting semantic code clones, outperforming CDLH with increases in F1 scores on BigCloneBench benchmarks [25]. 

Feng et al. in 2020, marked a significant advancement in transformer-based models by introducing CodeBERT, derived from the BERT and RoBERTa architectures [8]. By leveraging transformer-based neural networks, CodeBERT adeptly captures contextual embeddings, allowing for a nuanced understanding of similarities and variations within code fragments. Its bimodal pre-training in programming language and natural language has enabled it to excel in various downstream applications. In the realm of clone detection, CodeBERT shows promise by enhancing the accuracy of identifying code clones through its unique pretraining approach. It harnesses vast amounts of code data to recognize complex patterns that may escape traditional methods. 

Liu et al. introduced SSCD in 2022, a BERT-based clone detection method targeting Type 3 and Type 4 clones in large codebases [30]. SSCD computes embeddings for each code fragment using a BERT-based neural network, fine-tuned with contrastive loss, and identifies similar fragments through a k-nearest neighbor (k-NN) search. ### _E. DISCUSSION_ 

The results indicate that both GPT-3.5 Turbo and GPT-4 models perform competitively across all clone types. Notably, GPT-4 achieved a 5% improvement in precision compared to GPT-3.5 Turbo (Precision: 0.84 vs. 0.79) while maintaining consistent recall (0.91). This improvement highlights GPT4’s ability to better detect functional similarities across a broad spectrum of clone types, particularly Type-3 and Type4 clones, which are often challenging for traditional tools. While GPT-3.5 Turbo exhibited strong recall and balanced performance, GPT-4 demonstrated higher precision and F1 scores, making it more effective for analyzing complex semantic clones and intricate code relationships. 

Overall, both models performed well across all clone types, demonstrating the adaptability of our instructiontuned approach. A focused evaluation on Type-3 and Type-4 clones revealed the nuanced capabilities of GPT-4. Specifically, GPT-4 achieved a Precision of 0.93 and F1 Score of 0.93 for Type-4 clones, underscoring its ability to capture functional equivalence in code fragments that differ significantly in syntax. This focused performance highlights the effectiveness of instruction tuning in addressing deeper functional relationships, which traditional syntactic analysis methods often miss. 

A comparative analysis with existing tools further validates the strength of our approach. For overall clone detection (Type-1 to Type-4), GPT-4 achieved competitive metrics (Precision: 0.84, Recall: 0.91, F1 Score: 0.87) compared to tools like NiCad [2] and SourcererCC [32], which primarily focus on syntactic clones. For Type-3 and Type-4 clones, GPT-4 outperformed existing LLM-based tools such as CodeBERT [8] and CodeT5+ [39] in precision and F1 score, reinforcing its capability to handle semantic clones effectively. GPT-3.5 Turbo also demonstrated competitive performance, particularly in the recall, further showcasing the adaptability of instruction-tuned models. 

These findings demonstrate that instruction tuning effectively enhances task alignment by enabling models to detect complex semantic clones with minimal reliance on extensive labeled datasets. By guiding the model with task-specific prompts, our approach offers a scalable method to adapt general-purpose LLMs for specialized software engineering tasks. 

The integration of these models into IDE workflows further highlights their practical relevance. Real-time detection and automated refactoring within the IDE enable developers to identify and resolve code clones seamlessly. This practical contribution links advancements in clone detection research with the practical demands of software development workflows, positioning our approach as a scalable and actionable solution for modern software engineering challenges. 

### **VI. CONCLUSION** 

This paper introduces a novel methodology leveraging instruction-tuned large language models for effective code 

clone detection. Our approach applies few-shot instruction tuning to GPT-3.5 Turbo and GPT-4, enabling these general-purpose models to achieve competitive performance across all clone types, including the challenging complex (Type 3 and Type-4) clones. By tailoring task-specific prompts from the BigCloneBench dataset, we demonstrated the ability of instruction tuning to enhance functional understanding and adapt LLMs for code clone detection with minimal labeled examples. Experimental evaluations highlighted the robustness of our methodology, with the instruction-tuned GPT-4 surpassing both traditional tools and existing LLM-based approaches in detecting Type3 and Type-4 clones, achieving a notable balance of precision and recall. Furthermore, integrating these models into an IDE framework for real-time detection and automated refactoring underscores their practical applicability and scalability. This integration bridges theoretical advancements in LLMs with real-world software engineering workflows, enhancing developer productivity and software maintainability. Our findings emphasize the transformative potential of instruction-tuned LLMs in software engineering. This work lays the foundation for future research in leveraging LLMs to address complex programming tasks and extending this framework to other software engineering domains.
