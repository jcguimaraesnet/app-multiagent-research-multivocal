# Open Tree Search

[Permalink: Open Tree Search](https://github.com/Genentech/opentreesearch#open-tree-search)

Open Tree Search is a PUCT-based tree search algorithm for LLM-guided code evolution, built on top of [OpenEvolve](https://github.com/algorithmicsuperintelligence/openevolve). Tree search replaces OpenEvolve's nine-plus island-evolution hyperparameters with a single one that performs well out of the box.

The implementation follows the appendix of Google Research's [_An AI system to help scientists write expert-level empirical software_](https://arxiv.org/abs/2509.06503) ( [blog post](https://research.google/blog/accelerating-scientific-discovery-with-ai-powered-empirical-software/)).

## How It Works

[Permalink: How It Works](https://github.com/Genentech/opentreesearch#how-it-works)

- **TreeNode**: tracks `depth`, `visits`, `children_ids`, `parent_id`.
- **PUCT selection** (`ucb_expand`): picks parents using rank-normalized scores plus an exploration bonus.
- **Backpropagation**: child evaluation increments visit counts up to the root.
- **Single knob**: `puct_exploration_constant` (default `1.0`) controls the exploration–exploitation trade-off, higher values favor under-visited branches.

## Installation

[Permalink: Installation](https://github.com/Genentech/opentreesearch#installation)

```
uv sync
```

## Configuration

[Permalink: Configuration](https://github.com/Genentech/opentreesearch#configuration)

The only tree-search-specific setting is `database.puct_exploration_constant`; everything else follows the [OpenEvolve config format](https://github.com/algorithmicsuperintelligence/openevolve).

Add your LLM info to `config.yaml`:

```
llm:
  models:
    - name: "gemini-2.5-flash"
      api_base: "https://generativelanguage.googleapis.com/v1beta/openai/"
      api_key: "${LLM_API_KEY}"
```

Then `export LLM_API_KEY="..."` before running. The free Gemini tier works for the examples, get a key at [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey).

## Examples

[Permalink: Examples](https://github.com/Genentech/opentreesearch#examples)

### Function Minimization

[Permalink: Function Minimization](https://github.com/Genentech/opentreesearch#function-minimization)

A toy example that evolves an optimization algorithm for a 2D function with many local minima.

```
uv run treesearch-run examples/function_minimization/initial_program.py \
  examples/function_minimization/evaluator.py \
  --config examples/function_minimization/config.yaml \
  --output output/func_min/run_1/
```

Example evolved tree (rendered with the tools in [Visualizing a Run](https://github.com/Genentech/opentreesearch#visualizing-a-run)):

[![](https://github.com/Genentech/opentreesearch/raw/main/examples/function_minimization/visualization.png)](https://github.com/Genentech/opentreesearch/blob/main/examples/function_minimization/visualization.png)

Open [tree\_search\_view.html](https://github.com/Genentech/opentreesearch/blob/main/examples/function_minimization/tree_search_view.html) in a browser for the interactive version.

### Circle Packing (n=26)

[Permalink: Circle Packing (n=26)](https://github.com/Genentech/opentreesearch#circle-packing-n26)

Pack 26 circles in a unit square to maximize the sum of radii. [AlphaEvolve](https://arxiv.org/abs/2506.13131) reached 2.635 and [OpenEvolve](https://github.com/algorithmicsuperintelligence/openevolve/tree/main/examples/circle_packing) reached 2.634 in 200 iterations; we reach 2.632 in 121 iterations with `Qwen3-Coder-Next` using default settings.

[![](https://github.com/Genentech/opentreesearch/raw/main/examples/circle_packing/evolution.gif)](https://github.com/Genentech/opentreesearch/blob/main/examples/circle_packing/evolution.gif)[![evolution.gif](https://github.com/Genentech/opentreesearch/raw/main/examples/circle_packing/evolution.gif)](https://github.com/Genentech/opentreesearch/blob/main/examples/circle_packing/evolution.gif)[Open evolution.gif in new window](https://github.com/Genentech/opentreesearch/blob/main/examples/circle_packing/evolution.gif)

```
uv run treesearch-run examples/circle_packing/initial_program.py \
  examples/circle_packing/evaluator.py \
  --config examples/circle_packing/config.yaml \
  --output output/circle_packing/run_1/
```

## Visualizing a Run

[Permalink: Visualizing a Run](https://github.com/Genentech/opentreesearch#visualizing-a-run)

Each run writes checkpoints to `output/<run>/checkpoints/checkpoint_<N>/`. The tools below turn a checkpoint into a self-contained HTML page (open directly, no server).

### Optional: LLM summaries per node

[Permalink: Optional: LLM summaries per node](https://github.com/Genentech/opentreesearch#optional-llm-summaries-per-node)

Generates a per-program summary and parent-diff to `<checkpoint>/summaries/<program_id>.json`; the tree viewer auto-embeds them if present.

```
uv run python tools/generate_program_summaries.py \
  output/func_min/run_1/checkpoints/checkpoint_10 \
  --config examples/function_minimization/config.yaml
```

### Tree viewer

[Permalink: Tree viewer](https://github.com/Genentech/opentreesearch#tree-viewer)

Embeds the program tree, per-node metrics, and source. Re-run after summaries to include them.

```
uv run python tools/generate_tree_search_view.py \
  output/func_min/run_1/checkpoints/checkpoint_10
# writes <checkpoint>/tree_search_view.html
```
